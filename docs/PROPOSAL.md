## 1. エクシア様の現状と課題

### 現在の運用（アナログ）

- 日報・点呼記録・日常点検は**紙で管理**している
- アルコールチェックは実施しているが、**誰が確認したか記録されていない**
- 未提出者の把握は**管理者の記憶と目視**に依存
- ドライバーへの周知は**LINE公式アカウント**で行っている（社内報）
- データの保管は**紙のファイリング**（1年分、事故は3年分）

### 課題

| # | 課題 | リスク |
|---|------|--------|
| 1 | 令和7年4月の法改正に未対応 | **行政指導・処分の対象になる** |
| 2 | アルコールチェックの確認者が不明 | 法定の「点呼記録」として不十分 |
| 3 | 未提出を把握できない | 法定記録の欠落 |
| 4 | 紙のデータが散在 | 監査時に探す手間、紛失リスク |
| 5 | 管理に時間がかかる | 管理者が本来の業務に集中できない |

## 2. 令和7年4月 法改正のポイント

令和6年に法律が改正され、**令和7年（2025年）4月から**軽貨物運送事業者に対する安全対策が大幅に強化されました。

### 新たに義務化される項目（NEW）

| # | 義務化される内容 | 具体的にやること |
|---|----------------|---------------|
| 1 | **貨物軽自動車安全管理者の選任** | 営業所ごとに安全管理者を選任し、運輸支局に届出 |
| 2 | **安全管理者講習の受講** | 選任前に講習受講、その後2年ごとに定期講習 |
| 3 | **業務の記録** | 運転者名、車両番号、業務開始/終了日時・地点、走行距離、経過地点を記録し**1年間保存** |
| 4 | **事故の記録** | 事故発生時に概要・原因・再発防止策を記録し**3年間保存** |
| 5 | **初任運転者等への指導** | 初任者・65歳以上・事故者への特別指導と適性診断 |

### 引き続き必要な項目

| # | 内容 | 具体的にやること |
|---|------|---------------|
| 6 | **点呼** | 乗務前後にアルコールチェック、体調確認、車両状況確認。記録を**1年間保存** |
| 7 | **日常点検** | エンジンルーム、ライト、タイヤ、運転席周りの目視点検 |
| 8 | **勤務時間の遵守** | 年3,300時間、月284時間、1日13時間以内 |

### 守れないとどうなるか

- 運輸支局からの**行政指導**
- 繰り返しの場合、**事業停止命令**の可能性
- 事故発生時に「安全管理体制の不備」として**重い責任**を問われる

**→ このシステムは、上記の法定記録項目をデジタルで管理し、記録・保存・アラートを自動化します。**
※ 点呼（対面確認）そのものはシステム外で実施が必要です（§4 参照）。

## 3. ご提案するシステム

### システムの概要

**「LINEから入力するだけで、法律が求める記録がすべて自動で残るシステム」**

```
┌──────────────────────────────────────────────────────────┐
│                                                            │
│   ドライバー（LINE）          管理者（パソコン）           │
│                                                            │
│   📱 朝：出勤ボタンを押す     💻 提出状況を一覧で確認     │
│       → 点呼項目を入力         → 未提出者にはアラート     │
│       → 車両点検チェック       → 過去データを検索・閲覧   │
│                                → CSVダウンロード          │
│   📱 夕：退勤ボタンを押す                                 │
│       → 走行距離・配送先入力                              │
│                                                            │
│   ⚠️ 事故時：事故報告          📊 データは自動でクラウドへ │
│       → 写真も添付可能         → 1年/3年の保存期間管理   │
│                                                            │
└──────────────────────────────────────────────────────────┘
```

### ドライバーの操作（LINE）

ドライバーがやることは**4つだけ**です。すべてLINEのメニューからタップするだけ。

#### ① 業務前報告（毎朝 - 所要時間：約2分）

出勤ボタンを押すと、日時が自動で記録されます。その後、以下を入力します。

| 入力項目 | 入力方法 | 備考 |
|---------|---------|------|
| 使用車両 | 選択（前回と同じなら自動） | ナンバープレートが表示される |
| 配送先（予定） | テキスト入力 | 過去の入力履歴からも選べる |
| アルコールチェック | 「検知なし/あり」を選択 | 確認者の名前も記録 |
| 体調 | 「良好/やや不調/不調」を選択 | 不調の場合は詳細を記入 |
| 荷物の数 | 数字を入力 | |

#### ② 日常点検（毎朝 - 所要時間：約1分）

車両の13項目をチェックリスト形式で確認します。

| カテゴリ | 点検項目 | 操作 |
|---------|---------|------|
| エンジンルーム | オイル・冷却水・バッテリー | OK/要整備 |
| ライト類 | ヘッドライト・ウインカー・ブレーキランプ | OK/要整備 |
| タイヤ | 空気圧・溝・損傷 | OK/要整備 |
| 運転席周り | ミラー・シートベルト・ブレーキ・ステアリング | OK/要整備 |

全項目OKならワンタップで完了。異常があれば詳細を記入。

#### ③ 業務後報告（毎夕 - 所要時間：約2分）

退勤ボタンを押すと、日時が自動で記録されます。

| 入力項目 | 入力方法 |
|---------|---------|
| 走行距離 | 数字を入力（km） |
| 配送先（実際） | テキスト入力（例：市川市〇〇） |
| アルコールチェック | 「検知なし/あり」を選択 |
| 配送した荷物の数 | 数字を入力 |

#### ④ 事故報告（発生時のみ）

事故が起きた場合のみ使用します。

| 入力項目 | 入力方法 |
|---------|---------|
| 発生日時 | 自動 or 手入力 |
| 発生場所 | テキスト入力 |
| 事故の概要 | テキスト入力 |
| 原因 | テキスト入力 |
| 再発防止策 | テキスト入力 |
| 写真 | スマホで撮影して添付 |

### 管理者の操作（パソコン）

管理者がパソコンのブラウザでできることは以下の通りです。

| 機能 | できること |
|------|----------|
| **ダッシュボード** | 今日の提出状況を一目で確認。未提出者は赤色で表示 |
| **日報一覧** | ドライバー別・日付別に過去の記録を検索・閲覧 |
| **事故報告一覧** | 過去の事故記録を閲覧（3年間自動保存） |
| **ドライバー管理** | ドライバーの追加・編集・退職処理 |
| **車両管理** | 車両の追加・編集・整備中ステータス変更 |
| **CSVエクスポート** | データをExcelで開ける形式でダウンロード（監査対応） |
| **LINE一斉配信** | 全ドライバーまたは選択したドライバーにLINEで連絡 |

### 自動化される機能

管理者が何もしなくても、システムが自動で以下を行います。

| 機能 | 動作 |
|------|------|
| **朝のリマインド** | 毎朝8:00にドライバーへ「業務前報告を入力してください」とLINE通知 |
| **未提出アラート** | 9:30に業務前報告の未提出者へ再通知 + 管理者にも通知 |
| **夕方のリマインド** | 19:00に業務後報告の未提出者へ通知 |
| **管理者サマリー** | 毎朝10:00に「本日の提出状況：○名提出 / ○名未提出」を管理者へ通知 |
| **前回値の自動表示** | ナンバープレート等は前回と同じなら自動でセット |
| **保存期間管理** | 1年/3年の保存期間を自動管理。期限切れ前に通知 |

## 5. 技術的な特徴

### なぜLINE連携なのか

| 方法 | メリット | デメリット |
|------|---------|----------|
| 専用アプリ開発 | 自由度が高い | ドライバーがインストールしてくれない。開発費が高い |
| Webブラウザ | インストール不要 | 毎回URLを開くのが面倒。通知が送れない |
| **LINE連携（採用）** | **既に全員が使っている。通知も送れる。インストール不要** | LINE公式アカウントの設定が必要 |

エクシア様は既にLINE公式アカウントで社内報を配信されているため、ドライバーへの展開がスムーズです。

### データの安全性

| 項目 | 対応 |
|------|------|
| データの保管場所 | 日本リージョンのクラウドサーバー |
| 通信の暗号化 | HTTPS（SSL/TLS） |
| アクセス制御 | 管理者しか全データを見られない |
| バックアップ | 自動（毎日） |
| 組織間のデータ分離 | 完全分離（他社のデータと混ざらない） |

### 将来の拡張性

本システムは「他の運送会社にも同じシステムを提供できる」設計になっています。

| 拡張例 | 難易度 |
|--------|--------|
| 他の運送会社への導入 | 環境変数の切り替えのみ |
| 勤務時間の自動集計・超過アラート | 低（データは既にある） |
| GPS連携による配送ルート記録 | 中 |
| PDF帳票の自動生成（印刷用） | 低 |
| 多言語対応（外国人ドライバー向け） | 中 |

## 6. 開発体制

### AI活用による高品質・低コスト開発

本システムは、3つのAIエンジンによる合議制で開発・品質管理を行います。

```
┌─────────────────────────────────────────────────────┐
│              3AI合議制 開発プロセス                    │
│                                                       │
│  Claude Code ──── アーキテクチャ設計・実装            │
│       ↕                                               │
│  Codex ────────── コードレビュー（17項目×2パス）      │
│       ↕                                               │
│  Gemini ───────── セキュリティ・パフォーマンス検証    │
│                                                       │
│  3者の合意がなければコードを本番に出さない             │
│                                                       │
└─────────────────────────────────────────────────────┘
```

この体制により、通常の開発会社で必要な「PM 1名 + エンジニア3〜4名 × 2ヶ月」の工程を、大幅に短縮・高品質化しています。

### 3-AIコンセンサスレビュー結果（2026年2月12日実施）

設計・提案内容について3つのAIが独立にレビューを行い、以下の改善を反映済みです。

| # | 指摘事項 | 対応内容 |
|---|---------|---------|
| 1 | 監査ログの書き込み権限 | 認証済みユーザーのみに制限（不正書き込み防止） |
| 2 | 休憩時間の記録方法 | 自由記述→構造化データに変更（勤務時間の正確な集計に対応） |
| 3 | 点呼の法的位置づけ | システムは「記録の補助」であり、管理者による確認行為は別途必要と明記 |
| 4 | LINE紐付けの自動化 | QRコード/登録URL方式で、ドライバー自身が簡単に紐付け可能に |
| 5 | 通信不安定時の対策 | フォーム入力内容の自動一時保存機能を追加 |
| 6 | バックアップ強化 | Supabase標準に加え、外部ストレージへの定期エクスポートを追加 |
| 7 | 障害時の代替フロー | 管理者による代理入力機能を追加 |

**→ これらの改善により、セキュリティ・法令準拠・運用耐性がさらに強化されています。**

## 12. よくある質問

**Q. ドライバーは新しいアプリをインストールする必要がありますか？**
A. いいえ。LINEの友だち追加をするだけです。LINEを既にお使いであれば、追加のインストールは不要です。

**Q. スマホが苦手なドライバーでも使えますか？**
A. はい。操作は「ボタンを押す → 項目を入力する → 送信する」だけです。毎日変わらない項目（ナンバープレート等）は自動で入るため、実質的な入力は最小限です。

**Q. インターネットが使えない場所では使えませんか？**
A. LINEが使える環境であれば使用できます。4G/5G回線やWi-Fi環境があれば問題ありません。万が一、入力途中で通信が途切れても、入力内容は自動的に一時保存されるため、通信が復旧すれば入力をやり直す必要はありません。

**Q. LINEやシステムに障害が起きたらどうなりますか？**
A. 障害時には管理者が代理入力できる機能があります。ドライバーは管理者に電話で報告し、管理者が管理画面から入力する運用に切り替えることで、法令上の記録に空白期間を生みません。

**Q. データが消えることはありませんか？**
A. 日次の自動バックアップに加え、週次でSupabaseとは別のクラウドストレージにもデータを保管しています。万が一の障害時でもデータは復旧可能です。

**Q. 今使っている紙の書式はどうなりますか？**
A. CSVエクスポート機能で、紙と同等の情報をExcelやPDFで出力できます。監査時にはこれを印刷してお使いいただけます。

**Q. 途中でドライバーが増えたり減ったりした場合は？**
A. 管理画面から簡単にドライバーの追加・削除ができます。追加費用はかかりません。

**Q. 他のシステム（運行管理、給与計算等）と連携できますか？**
A. 基本機能には含まれていませんが、CSVエクスポートにより他システムへのデータ取り込みは可能です。API連携が必要な場合は個別にご相談ください。
